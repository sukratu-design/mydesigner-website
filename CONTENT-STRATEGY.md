## Publishing Cadence

### Recommended: 2x per week (Tuesday + Thursday)

**Why not daily (current pace):**
- Google needs time to crawl and assess each post. Daily publishing on a low-authority domain dilutes signals.
- No time to promote each post (social, newsletters, communities).
- Reader fatigue -- subscribers ignore daily emails.

**Why not weekly:**
- You have 10+ drafts queued. 2x/week builds the content library faster while giving each post breathing room.
- Faster path to topical authority in target clusters.

## Conversion Tracking

### GA4 Event: `book_a_call`

Added to `js/main.js` -- fires on every click to `calendar.app.google` links sitewide.

**Event parameters:**
- `event_category`: "conversion"
- `event_label`: Button text (e.g., "Book a call", "Book a call -- it's free")
- `page_location`: Page path where click happened

**To complete setup:**
1. Go to GA4 Admin > Events > mark `book_a_call` as a conversion (key event)
2. Optionally create a second event for pricing page visits (`view_pricing`)

## Distribution Strategy

Publishing alone won't drive traffic. Each post needs distribution:

1. **LinkedIn** -- Share each post with a 3-4 line hook. Tag relevant topics. This is where SaaS founders and design leads live.
2. **Twitter/X** -- Thread format: pull 3-4 key insights from the post as a thread, link to full post at the end.
3. **Indie Hackers / Hacker News** -- For the more opinionated/data-driven posts (design ROI, cost breakdowns, case studies).
4. **Design communities** -- Dribbble, Behance (for case studies with visuals), Designer Hangout Slack.
5. **Email newsletter** -- Build a subscriber list. Even 50 engaged subscribers who forward to founders is more valuable than 5,000 pageviews.
